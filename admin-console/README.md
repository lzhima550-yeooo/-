# 四季夏木后台（一期最小版）

该目录是独立后台模块，只新增文件，不修改现有前端/后端业务代码。

## 一期范围

- 本地账号登录（演示级入口保护）
- 运行总览（健康、运行态、候选与帖子数量）
- 社区巡检（帖子列表检索 + 详情查看）
- 知识回流（候选筛选、审核记录查看、通过/驳回/回滚操作）
- 本地操作审计日志（浏览器端）

## 目录结构

- `index.html`：后台页面结构
- `admin.css`：后台样式（与主应用风格一致）
- `admin.js`：后台交互逻辑
- `server.mjs`：独立启动服务（静态托管 + `/api` 反向代理）

## 启动方式

1. 在项目根目录启动 API（端口 8787）：

```bash
npm run dev:api
```

2. 在 `admin-console` 目录启动后台服务（默认端口 6060）：

```bash
node server.mjs
```

3. 浏览器访问：

- `http://127.0.0.1:6060`

## 默认账号

- `superadmin / admin123`
- `reviewer / review123`
- `ops / ops123`

## 可选环境变量

- `ADMIN_HOST`：后台服务监听地址，默认 `127.0.0.1`
- `ADMIN_PORT`：后台服务端口，默认 `6060`
- `ADMIN_API_TARGET`：代理目标地址，默认 `http://127.0.0.1:8787`

示例（允许手机同局域网访问后台）：

```bash
ADMIN_HOST=0.0.0.0 node server.mjs
```

然后用同一局域网设备访问：`http://<你的电脑局域网IP>:6060`

## 验收建议（一期）

- 打开后台，能登录并进入三个页面
- 运行总览可正常读取 `/api/health`
- 社区巡检可加载帖子列表并查看详情
- 知识回流可完成 `approve / reject / rollback`，并看到 `/reviews` 更新
- 刷新后会话与 API Base 配置可恢复
